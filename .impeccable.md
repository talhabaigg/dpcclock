# Design Context

## Users
**Mixed office + field use**: Office staff during traditional work hours (desk-based) AND site managers/field workers who need to access the interface on mobile and tablet while on-site. This means the interface must work fluidly across desktop, tablet, and mobile without losing functionality.

**Context**: Daytime use (mostly), but with increasing mobile/field work requiring dark mode support for outdoor and low-light environments.

**Job to be done**: Create purchase requisitions quickly and accurately. Users need confidence in the process and clarity about what happens next.

## Brand Personality
**Approachable & Clear** — Friendly, straightforward, reduces anxiety.

- 3-word voice: **Clear, efficient, trustworthy**
- The goal is to make requisition creation feel straightforward, not intimidating or corporate
- Users should feel guided, not lost
- Avoid corporate coldness; embrace clarity over complexity

## Aesthetic Direction

### Current State Problems
- **Generic appearance**: Feels templated, like "default Shadcn" with no distinctive point of view
- **Unclear visual hierarchy**: Hard to tell what's important or what to do first
- **Cramped layout**: Too much information visible at once, overwhelming
- **Dated typography/color**: Current choices (even if technically modern) lack personality and feel generic

### Target Direction
Choose a design that is:
- **Memorable**: Not another beige corporate dashboard. Have a clear aesthetic POV.
- **Clear**: Strong visual hierarchy that guides users through the requisition flow
- **Breathing room**: Smart use of whitespace to reduce cognitive load
- **Distinctive typography**: Instrument Sans is already a good choice; use it with intentional hierarchy and personality
- **Modern color system**: Leverage OKLCH colors to create cohesive, intentional palettes (not generic brand colors)
- **Supports both light & dark modes** without feeling like a toggle—each theme should feel intentional

### Aesthetic Inspiration (What NOT to be)
- **Anti-references**: Generic admin dashboards, default Shadcn templates, corporate websites with side-stripe accent borders
- **Avoid**: gradient text, glassmorphism, rounded rectangles with generic shadows, dense card grids

### Aesthetic Inspiration (What TO be)
- **Idea**: Make this feel like a well-designed workflow tool—something that respects the user's time and intelligence
- **Example feeling**: Think along the lines of Vercel's dashboard (purposeful, minimal, clear hierarchy) or Linear's interface (fast, direct, uses space intentionally)

## Design Principles

1. **Hierarchy over complexity** — One primary action per context. Secondary actions are discoverable but not distracting.
2. **Clarity through restraint** — Use spacing, typography scale, and color strategically. Don't use everything at once.
3. **Field-friendly** — Mobile/tablet must work as a first-class citizen, not a shrunk desktop version. Adapt, don't amputate.
4. **Accessibility by default** — Support both light and dark modes. Use perceptually uniform color (OKLCH). Test keyboard navigation.
5. **Intentional aesthetic** — This is not a generic dashboard. Every design choice should have a reason.

## Current Tech Stack
